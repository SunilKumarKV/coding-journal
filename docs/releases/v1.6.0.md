# coding-journal v1.6.0 — Multi-platform Capture & Import Automation

## Summary

coding-journal v1.6.0 expands the journal from a mostly LeetCode-oriented workflow into a multi-platform capture and import system. This release adds Codeforces imports, CodeChef profile summaries, HackerRank imports, and the generated data updates needed to power a multi-platform coding profile downstream.

## Highlights

- Added multi-platform coding profile support in generated journal data
- Added Codeforces metadata import for accepted submissions without fake code
- Added CodeChef profile summary fallback when only solved-count data is available
- Added HackerRank import support for public recent challenge metadata
- Refreshed generated data so raw URLs and metadata resolve from `main`

## Features

- Multi-platform normalized problem records for:
  - LeetCode
  - Codeforces
  - CodeChef profile summaries
  - HackerRank
- Import pipeline improvements for metadata-only submissions and grouped accepted records
- Journal data builders now include platform summary records alongside normal problem records
- CLI sync flow now regenerates release-ready `data/problems.json`, `data/projects.json`, `data/stats.json`, and `data/metadata.json`
- Added new generated problem records for HackerRank and CodeChef profile-summary data

## Fixes

- Normalized generated `detailUrl` and repository metadata to point at `main` for release output
- Improved platform casing and metadata handling in generated journal data
- Preserved no-fake-code behavior for imports where only metadata is available
- Strengthened tests around platform import, sync diagnostics, and multi-language submission handling

## Build/Test Status

- `npm test`: passed again on `main` after merge, `32/32`

## Known Limitations

- Several existing LeetCode entries are still unverified and were reported by `cj sync`
- Some imported records are intentionally metadata-only until code is attached later
- `docs/releases/latest.md` is still a rolling snapshot and may not match a tagged release exactly without a dedicated publish step
